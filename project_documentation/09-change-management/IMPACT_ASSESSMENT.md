# Impact Assessment
## NextSaaS - Change Impact Analysis

**Last Updated:** December 23, 2025  
**Version:** 2.0.0

---

## Overview

This document describes how to assess the impact of changes to NextSaaS.

---

## Impact Dimensions

### 1. Technical Impact

**Factors**:
- Code changes required
- Database changes
- API changes
- Breaking changes

**Assessment**: Low | Medium | High

---

### 2. User Impact

**Factors**:
- User-facing changes
- Workflow changes
- Feature removals
- Performance changes

**Assessment**: Low | Medium | High

---

### 3. Security Impact

**Factors**:
- Security implications
- Vulnerability risks
- Compliance impact
- Access control changes

**Assessment**: Low | Medium | High

---

### 4. Performance Impact

**Factors**:
- Response time changes
- Throughput changes
- Resource usage
- Scalability impact

**Assessment**: Low | Medium | High

---

## Impact Assessment Process

### 1. Identify Changes

**List**:
- All code changes
- Database changes
- Configuration changes
- Dependency changes

---

### 2. Analyze Impact

**For Each Change**:
- Technical impact
- User impact
- Security impact
- Performance impact

---

### 3. Risk Assessment

**Identify Risks**:
- Breaking changes
- Data loss risks
- Security risks
- Performance risks

---

### 4. Mitigation Planning

**Plan**:
- Testing strategy
- Rollback plan
- Communication plan
- Monitoring plan

---

## Impact Matrix

### Low Impact

**Characteristics**:
- Internal changes only
- No user-facing changes
- No breaking changes
- Low risk

**Process**: Standard PR process

---

### Medium Impact

**Characteristics**:
- Some user-facing changes
- Minor breaking changes
- Moderate risk

**Process**: Design review + PR process

---

### High Impact

**Characteristics**:
- Major user-facing changes
- Breaking changes
- High risk
- Architecture changes

**Process**: Full design review + approval + staged rollout

---

## Assessment Checklist

### Before Change

- [ ] Impact assessed
- [ ] Risks identified
- [ ] Mitigation planned
- [ ] Stakeholders notified
- [ ] Testing planned

---

### After Change

- [ ] Impact verified
- [ ] Risks monitored
- [ ] Performance checked
- [ ] User feedback collected
- [ ] Documentation updated
